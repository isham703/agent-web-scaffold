# MCP Server Setup Guide

This project uses 8 MCP servers to give AI agents fast, accurate code navigation. This guide covers installation, configuration, and verification.

## Quick Start

```bash
# Install prerequisites (see below), then:
npm run setup-mcp
```

This detects installed tools, generates MCP configs for **both** Claude Code and Kiro, sets up git hooks, and runs the initial Zoekt index. All generated configs are gitignored — each developer (or agent) runs this once on their machine.

The setup script will **refuse to run** if `.gitignore` doesn't protect the generated files.

## Local-Only Policy

MCP configs contain machine-specific absolute paths and are **never committed to git**:

| File | IDE | Gitignored | Generated by |
|------|-----|------------|-------------|
| `.mcp.json` | Claude Code | Yes | `npm run setup-mcp` |
| `.kiro/settings/mcp.json` | Kiro | Yes | `npm run setup-mcp` |
| `CLAUDE.local.md` | Claude Code | Yes | `npm run setup-mcp` (from template) |

The setup script (`scripts/setup-mcp.js`) is committed. The configs it generates are not. **Do NOT `git add` these files.**

## Overview

| Server | Purpose | Language-specific? | Install effort |
|--------|---------|-------------------|----------------|
| **TypeScript LSP** | Symbol definitions, references, hover, rename | Yes (TypeScript) | 2 min |
| **Zoekt** | Indexed code search with regex, file/language filters | No (indexes any Git repo) | 5 min |
| **Scantool (Tree-Sitter)** | Structural pattern search (components, hooks, types) | No (20+ languages) | 1 min |
| **Knowledge RAG** | Semantic search over Documentation/ | No (reads markdown) | Already installed* |
| **Mind** | Unified cross-system search | No (wraps other backends) | Already installed* |
| **Feature Metadata** | Queries feature.yaml + component_catalog.yaml | No (reads YAML) | Already installed* |
| **Context Bundle** | Session state persistence | No (stores YAML) | Already installed* |
| **Change-Impact** | Blast radius analysis | No (reads YAML/markdown) | Already installed* |

*If you already have `minimal-server.py` from another project. Otherwise see [Minimal Server Setup](#minimal-server-setup).

## Prerequisites

- **Node.js 18+** and npm
- **Go 1.21+** (for Zoekt)
- **Python 3.11+** (for minimal-server.py)
- **Ollama** with `nomic-embed-text` model (for Knowledge RAG)
- **mcp-language-server** Go binary (for TypeScript LSP)

## 1. TypeScript LSP

Provides: `definition`, `references`, `hover`, `diagnostics`, `rename_symbol`, `edit_file`

### Install

```bash
# TypeScript language server (speaks LSP over stdio)
npm install -g typescript typescript-language-server

# mcp-language-server (wraps any LSP as MCP tools)
go install github.com/isaacphi/mcp-language-server@latest
```

### Requirements

- `tsconfig.json` must exist in project root (already included in scaffold)
- `node_modules` must be installed (`npm install`)
- No build step needed — works directly from source files

### Verify

```bash
# Check both are on PATH
which typescript-language-server  # should print a path
which mcp-language-server         # should print a path
```

### Known Limitations

- May fail on very large files (1800+ lines) — use `hover` as fallback
- Single workspace — for monorepos, point at root with project references

### Alternative: ts-language-mcp

For richer TypeScript-specific intelligence (25+ tools including AST search, call hierarchy):

```json
{
  "typescript": {
    "command": "npx",
    "args": ["ts-language-mcp", "."]
  }
}
```

Trade-off: more tools but separate from the uniform `mcp-language-server` interface.

## 2. Zoekt (Indexed Code Search)

Provides: `search` (regex with file/language filters), `list_repos`

### Install

```bash
# Zoekt binaries
go install github.com/sourcegraph/zoekt/cmd/zoekt-git-index@latest
go install github.com/sourcegraph/zoekt/cmd/zoekt-webserver@latest

# Optional: better symbol ranking
brew install universal-ctags

# MCP wrapper (zero dependencies, ~60 lines)
git clone https://github.com/cjlludwig/zoekt-mcp.git ~/.local/mcp-servers/zoekt-mcp
```

### Initial Index

```bash
# Index the repo
zoekt-git-index -index ~/.zoekt/index -branches main /path/to/this/project

# Start web server (background)
zoekt-webserver -index ~/.zoekt/index -listen :6070 &
```

### Auto-Reindex

Run `bash scripts/install-hooks.sh` to install git hooks that reindex on every commit/merge.

### Verify

Open `http://localhost:6070` in browser and search for `lang:TypeScript`.

### Web-Specific Query Tips

| Query | What it finds |
|-------|--------------|
| `lang:TypeScript import.*from` | All TypeScript imports |
| `lang:TSX function\s+\w+.*Props` | React components with props |
| `f:src/.*\.tsx? useEffect` | useEffect usage in source files |
| `-f:.*\.test\. useState` | useState excluding test files |
| `f:hooks/ use[A-Z]` | Custom hooks in hooks directories |
| `sym:useAuth` | Symbol search (requires ctags) |

**Note:** `.tsx` files are `lang:TSX`, not `lang:TypeScript` — they are separate in Zoekt.

## 3. Scantool (Structural Search)

Provides: `scan_file`, `scan_directory`, `search_structures`, `preview_directory`

### Install

```bash
# Zero-install via uvx (requires uv)
# Nothing to pre-install — uvx runs it on demand

# Or add to Claude Code directly:
claude mcp add scantool -- uvx scantool
```

### Useful Queries

- Find all React components: `search_structures` with type filter "function", name pattern `[A-Z]*`
- Find all hooks: name pattern `use[A-Z]*`
- Find all exported types: type filter "type" or "interface"
- Find by decorator: decorator filter `@Component`, `@Injectable`, etc.

### Alternative: treesitter-mcp (Rust, real parsing)

For true tree-sitter S-expression queries:

```bash
git clone https://github.com/Christoph/treesitter-mcp.git ~/.local/mcp-servers/treesitter-mcp
cd ~/.local/mcp-servers/treesitter-mcp && cargo build --release
```

```json
{
  "treesitter": {
    "command": "~/.local/mcp-servers/treesitter-mcp/target/release/treesitter-mcp",
    "args": []
  }
}
```

## 4. Minimal Server (5 identities)

The remaining 5 servers all run from a single Python file: `~/.local/mcp-servers/minimal-server.py`

### Minimal Server Setup

If you don't have `minimal-server.py` from another project:

```bash
# Create directory
mkdir -p ~/.local/mcp-servers/knowledge-rag/vectordb/web-scaffold

# Copy minimal-server.py from another project or install from source
# (See BibleStudy project for reference implementation)
```

### Knowledge RAG Index

```bash
# Ensure Ollama is running with the embedding model
ollama pull nomic-embed-text
ollama serve &

# Initial index (from project root)
python3 ~/.local/mcp-servers/minimal-server.py knowledge --reindex
```

### Multi-Project Support

Each project needs a separate Knowledge RAG index. Set `KNOWLEDGE_INDEX_PATH` env var per project in `.mcp.json`. If `minimal-server.py` doesn't support this yet, patch line ~867:

```python
# Change from:
_INDEX_PATH = Path.home() / ".local" / "mcp-servers" / "knowledge-rag" / "vectordb" / "embeddings.json"

# To:
_INDEX_PATH = Path(os.environ.get("KNOWLEDGE_INDEX_PATH",
    str(Path.home() / ".local" / "mcp-servers" / "knowledge-rag" / "vectordb" / "embeddings.json")))
```

Also patch `_REINDEX_LOCK` to be relative to the index path:
```python
_REINDEX_LOCK = _INDEX_PATH.parent / ".reindex.lock"
```

### Verify All Servers

```bash
# Quick smoke test — each should respond without error
# (Run from project root)
echo '{"method":"tools/list"}' | python3 ~/.local/mcp-servers/minimal-server.py knowledge
echo '{"method":"tools/list"}' | python3 ~/.local/mcp-servers/minimal-server.py mind
echo '{"method":"tools/list"}' | python3 ~/.local/mcp-servers/minimal-server.py feature-metadata
echo '{"method":"tools/list"}' | python3 ~/.local/mcp-servers/minimal-server.py context-bundle
echo '{"method":"tools/list"}' | python3 ~/.local/mcp-servers/minimal-server.py change-impact
```

## Search Ladder (Priority Order)

When searching for code or information, use this tool priority:

| Priority | Task | Tool |
|----------|------|------|
| 0 | General question about past work/decisions | Mind `mind_search` |
| 1 | Know the symbol name | LSP `definition` |
| 2 | Need all usages | LSP `references` |
| 3 | Structural patterns (components, hooks, types) | Scantool `search_structures` |
| 4 | Code with file/language filters | Zoekt `search` |
| 5 | Find files by name | Zoekt `search` or `Glob` |
| 6 | Conceptual/architectural info | Knowledge `knowledge_search` |
| 7 | Feature scope/file list | Feature Meta `feature_get` |
| 8 | Service metadata | Feature Meta `component_get` |
| 9 | Simple text in known file | `Grep` |
| 10 | Blast radius | Change-Impact `impact_query` |

## Troubleshooting

| Problem | Fix |
|---------|-----|
| LSP returns stale results | Run `npm install` then restart language server |
| Zoekt finds nothing | Run `zoekt-git-index` manually, check `zoekt-webserver` is running |
| Knowledge RAG empty results | Check Ollama is running: `ollama serve &`, then reindex |
| Feature metadata empty | Verify `Documentation/features/*/feature.yaml` exists |
| MCP server not responding | Kill and let framework respawn: `pkill -f 'minimal-server.py <identity>'` |
